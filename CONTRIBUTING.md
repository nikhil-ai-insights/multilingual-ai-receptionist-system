# 🤝 Contributing to Multilingual AI Receptionist

First of all, **thank you for considering contributing!** 🚀  
Your contributions help improve this project and make it more useful for everyone.

---

## 📌 How You Can Contribute

There are many ways to get involved:

| Type | Description |
|---|---|
| 🐛 **Bug Reports** | Found something broken? Let us know |
| 💡 **Feature Suggestions** | Have a great idea? We'd love to hear it |
| 🛠️ **Workflow Improvements** | Help optimize or extend existing flows |
| 📄 **Documentation** | Improve guides, README, or inline comments |
| ⚡ **Performance** | Make things faster and more efficient |
| 🌍 **Multilingual Support** | Add support for new languages |

---

## 🧑‍💻 Getting Started

### 1. Fork the Repository

Click the **Fork** button at the top right of this page to create your own copy.

### 2. Clone Your Fork

```bash
git clone https://github.com/your-username/multilingual-ai-receptionist.git
cd multilingual-ai-receptionist
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

> Use descriptive branch names like `feature/whatsapp-integration` or `fix/calendar-sync-bug`

## 📂 Project Structure Rules

| Directory | Purpose |
|---|---|
| `/workflows` | All new workflow JSON files go here |
| `/assets` | Images, diagrams, and visual resources |
| `/docs` | Documentation, architecture notes, guides |

---

## ✅ Commit Guidelines

Write **clear and descriptive** commit messages following this convention:

```
feat: add appointment reminder system
fix: resolve calendar sync issue
docs: update README with setup steps
refactor: simplify booking flow logic
test: add test case for cancellation flow
```

| Prefix | When to Use |
|---|---|
| `feat` | A new feature |
| `fix` | A bug fix |
| `docs` | Documentation changes |
| `refactor` | Code restructuring without feature change |
| `test` | Adding or updating tests |

---

## 🚀 Submitting a Pull Request

1. **Push your branch** to GitHub:

```bash
git push origin feature/your-feature-name
```

2. **Open a Pull Request** on the original repository
3. In your PR description, include:
   - ✏️ Clear description of the changes made
   - 📸 Screenshots (if UI or workflow related)
   - 🧪 Steps to test your changes

---

## 🔍 Code Review Process

- Your PR will be reviewed by the maintainer
- Constructive feedback may be provided
- Changes might be requested before merging
- Once approved, your contribution will be merged 🎉

---

## 🐞 Reporting Issues

When reporting a bug, please include the following:

- **Description** — What went wrong?
- **Steps to Reproduce** — How can we trigger the issue?
- **Expected Behavior** — What should have happened?
- **Screenshots** — Attach visuals if applicable

> Open an issue here → [GitHub Issues](https://github.com/nikhil-ai-insights/multilingual-ai-receptionist/issues)

---

## 💡 Feature Requests

Have an idea for a new feature? Here's how to propose it:

1. **Explain the feature** clearly and concisely
2. **Mention the use case** — who does it help and how?
3. **Suggest an implementation** approach (optional but appreciated)

> Open a feature request → [GitHub Issues](https://github.com/nikhil-ai-insights/multilingual-ai-receptionist/issues)
