# 🔐 Security Policy

Thank you for helping keep **Multilingual AI Receptionist** safe and secure.  
We take security seriously and appreciate responsible disclosure from the community.

---

## 📌 Supported Versions

We actively maintain and provide security updates for the following versions:

| Version | Supported |
|---|---|
| Latest | ✅ Yes |
| Older versions | ❌ No |

> Always use the latest version to ensure you have the most up-to-date security patches.

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it **responsibly and privately** — do not open a public issue.

### 📩 How to Report

- 🔒 **Preferred:** Create a [private security report](https://github.com/nikhil-ai-insights/multilingual-ai-receptionist/security/advisories/new) via GitHub Security Advisories
- 💬 **Alternative:** Contact the maintainer directly via GitHub → [@nikhil-ai-insights](https://github.com/nikhil-ai-insights)

### 📝 Please Include

To help us investigate quickly, provide as much detail as possible:

| Field | Details |
|---|---|
| **Description** | Clear explanation of the vulnerability |
| **Steps to Reproduce** | How to trigger the issue |
| **Possible Impact** | What could be affected or exploited |
| **Proof of Concept** | Screenshots, logs, or PoC code (if applicable) |

---

## ⚠️ Responsible Disclosure

We follow a responsible disclosure process. Please:

- 🚫 **Do not** publicly disclose the issue before it has been fixed
- ⏳ **Give us** reasonable time to investigate and release a patch
- 🛑 **Avoid** exploiting the vulnerability in any way
- 🤝 **Work with us** collaboratively to resolve the issue

---

## ⏱️ Response Timeline

Once a report is received, we aim to:

| Milestone | Timeframe |
|---|---|
| Acknowledge receipt | Within **48 hours** |
| Provide initial assessment | Within **3–5 business days** |
| Fix critical vulnerabilities | As soon as possible |
| Notify reporter of resolution | Upon patch release |

---

## 🔒 Security Best Practices

If you're **using or contributing** to this project, follow these practices:

- 🔑 **Keep API keys secure** — never commit secrets to the repository
- 🌿 **Use environment variables** for all credentials and sensitive config
- 🔐 **Enable authentication** wherever applicable
- ✅ **Validate all external inputs** to prevent injection attacks
- 🌐 **Use HTTPS** for all API and service integrations
- 🔄 **Regularly update dependencies** to patch known vulnerabilities

---

## 🛡️ Scope

This security policy applies to the following components of the project:

- 🤖 AI workflow automation engine
- 🔗 API integrations (OpenAI, Google Calendar, Twilio SMS, Email/SMTP)
- 🗄️ Data storage (Google Sheets / Database layer)
- 🎙️ Voice and communication systems (TTS/STT)

---

## 🚫 Out of Scope

The following are **not** considered valid security issues under this policy:

- 📧 Spam or social engineering attacks
- 🔧 Vulnerabilities in third-party services (OpenAI, Twilio, Google, etc.)
- ⚙️ Misconfiguration on the user's own setup or environment
- 🌐 Issues unrelated to this project's codebase or workflows
